  Microcorruption: Santa Cruz

        April 20, 2022

    Hello again hacker fam! This is part one of a series of hardware CTF challenges brought to you by Microcorruption.  Microcorruption is a wargame site that specializes in low-level hardware debugging.  I highly recommend creating an account for practice.  You will also need to learn the basics of the MSP430 microcontroller, which is the device that controls the challenge locks.  View their About page for more information.

This challenge is a bit more intermediate and assumes some basic knowledge of assembly language, stack layout, and the memory dump window in regards to brevity.  Earlier Microcorruption challenges have featured only 1 user input prompt before continuing with reverse engineering; however, the Santa Cruz room prompts the user for both the username and the password.  This will be important later.  The first step is to input dummy input and observe program activity as a result.  Enter 'c' to start the program and enter '123456789abcdef' as the username input


    Enter 'f' twice to return from the signal interrupt and getsn functions and return to the main logic function.  The rest of the program continues as shown below.  There is a bit more important content when scrolling down, but the jump instructions become important shortly.  Inspecting the assembly listing, there are two calls to strcpy.  


   Inspecting the assembly listing, there are two calls to strcpy.  The first call copies the username input from one address (2404) to another (43a2).  Respective parameters are stored in r14 and r15 for source and destination addresses.  Notice the location around address 43a2 is unpopulated before the call.


   Then the username input is moved after the strcpy call.


   Shortly afterwards, another call is made to getsn to get the password.  Since this is the first pass of the program, enter the alphabet in all uppercase as the password and hit send.  Hit 'f' twice to get back to the main login function.


   Another strcpy function call follows, which moves the user-supplied password to another location in memory.  This time the string is copied to a location just after the username destination (43b5).
