  Quick Primer

    A brief introduction to the Caeser cipher is needed to understand the Vigenere cipher.  The Caeser cipher uses the alphabet as its list of possible values and one number as its key.  So a phrase "I love dogs" with a key of 3 will shift each letter of "ILOVEDOGS" by three to make "LORYHGRJV".  If a letter is shifted after 'Z', it will start from the beginning of the alphabet.  So a character 'Y' with the same key will shift by 3 to make 'B'.  This is a very simple mechanism that is scaled in the Vigenere cipher.
    The Vigenere cipher is an advancement of the Caeser cipher in that it uses a word or phrase as its key.  Instead of shifting every letter in "I love dogs" by a simple number, each letter is shifted based on the letter of the key it lines up with.  Let's say the key is "hello", each letter of "hello" would be superimposed on the phrase "I love dogs" and shifted based on the letter value of the key:

I L O V E D O G S H E L L O H E L L + ----------------- P P Z G S K S R D

    In the above, 'I' has a value 8 (starting at index 0) and 'H' has a value of 7.  Adding the two values gives 15, which is letter 'P'.  Repeat for the remaining numbers to get string P P Z G S K S R D. This string is the ciphertext.  Note that a sum greater than 25 must be modular divided by 26 due to letter shift.  For example, V (21) + L (11) = 32 and 32 mod 26 = 6 which is 'G'.  To decrypt the ciphertext, subtract the key from the ciphertext.

P P Z G S K S R D H E L L O H E L L (-) ----------------- I L O V E D O G S

    Two functions are required with two inputs each: one for encryption and one for decryption, both use the same key as input.  As seen in the Quick Primer section, it is clear the decryption function will be the reverse of encryption.  In order to apply encryption, we need to enumerate both the plaintext and key.  Enumeration requires a mapping of each letter to its integer value so a Python dictionary works perfectly.  Blank lists are initialized for both the plaintext and Vigenere key as plain_keys and vigenere_keys respectively.  I apologize in advance for the name confusion.  Each list is filled using a nested for-loop to iterate over each key-value pair in character_key while iterating over each letter in the plaintext or Vigenere key.  If the value is a match, it is appended to its respective list: plain_keys or vigenere_keys.  Below is the code generated so far.

    Once the plaintext and Vigenere key are loaded into lists, both must be the same length in order to execute the cipher.  This can be checked with an if-statement to see if vigenere_keys is shorter than plain_keys.  If the opposite is true, no repetition of the Vigenere key is required and operation will stop at the end of the plaintext.  A discrete number of iterations is calculated to determine how many times the Vigenere key will cleanly fit into the plaintext.  This number is expressed by (length of plaintext - (length of plaintext mod length of Vigenere key)) / length of Vigenere key.  It is important to note the number of iterations starts from 0 to account for the remainder of the Vigenere key that must be filled.  This means if iterations = 0 is calculated, at least one loop will occur because that means the plaintext and Vigenere key are the same length thanks to the if-statement.  The inner loop cycles through each character of vigenere_keys and appends it to its own list.  This occurs for as many iterations as expressed by the outer loop until the end of plaintext is reached.  Below is the code excerpt. 

													
# These are required calculations prior to extending vigenere_keys
len_plain = len(plain_keys)
len_vig = len(vigenere_keys)
mod = len_plain % len_vig

# Extend vigenere_keys to match plain_keys in length.  Check if
# vigenere_keys is shorter than plain_keys
if len_vig < len_plain:
  iterations =  (len_plain - mod) / len_vig

  for iter in range(0, int(iterations)):
    for j in range(0, len_vig):
      # Exit loop when end is reached
      if len(vigenere_keys) == len(plain_keys):
         break
      else:
         vigenere_keys.append(vigenere_keys[j])

   We're almost there, Tim!  With plaintext and extended Vigenere key the same length, simply add their values as described in the Quick Primer section.  Don't forget to modular divide by 26!  A new list, encrypted_numbers holds the result of each character.  Then use the dictionary key to convert from values to characters similar to what was done at the beginning.  Then return the encrypted string: encrypted. 

												
# Add the elements of each array together and mod 26.  If len(vigenere) is
# less than len(plaintext), restart at begining of key.  If len(vigenere)
# is greater than len(plaintext), stop at end of plaintext
# Result is encrypted_numbers list
encrypted_numbers = []
for i in range(0, len(plain_keys)):
  new = (plain_keys[i] + vigenere_keys[i]) % 26
  encrypted_numbers.append(new)

# Convert enumerated plaintext + key into ciphertext string by referencing
# the character_key dictionary
# Initialize empty string
encrypted = ""
for number in encrypted_numbers:
  for key, value in character_key.items():
    if key == number:
      encrypted += value

# Result of encrypt()
return encrypted

  Bringing It Together

    Finally, everything above can be placed into the body of function def encrypt(plaintext, vigenere):; except for the dictionary because it is globally defined.  The decrypt() function is basically the same as encrypt() except backwards and uses the ciphertext instead of the plaintext. 

													
# Encrypt plaintext
crypto = encrypt(plaintext, vigenere)
print("\n####################################")
print("\nEncrypted text: " + crypto)
print("\n####################################")

  Breaking the Cipher

    Encryption and decryption are interesting, but breaking the cipher is what matters.  This can be done using exhaustive search (brute force) or statistical analysis methods.  Exhaustive search is useful when the key size is small enough.  Statistical analysis is interesting because it can reveal key length when used correctly; however in this assignment, the key length is known so sophisticated statistical analysis is not required.  If interested in decrypting ciphertext WITHOUT knowing key length, Al Sweigart's book walks through a more comprehensive approach in chapter 18.  Regardless of sophistication, both statistical analysis procedures require frequency analysis of the key characters.  Certain letters occur more frequently in the English language than others.  Below is a frequency table expressed in decimal form.
