Root-me Cracking Challenges (Part 1)

This is part one of a series in reverse engineering binaries provided by root-me.org. Root-me is a French based computer security community that emphasizes hands-on learning. I highly recommend creating an account for practice. If you are new to the field, you should read their Frequently Asked Questions (FAQ) page for a "learning path." It includes links to textbooks, research papers, and relevant challenges.

Since this is part 1 of the cracking challenges, a short primer on binary analysis is needed. The purpose of binary analysis is to explore the behavior of executable files when source code may not be available. This is typically done at the assembly code level but can involve Intermediate Representations (IRs) on a tool-by-tool basis. Most Linux machines ship with basic binary analysis tools such as file, readelf, objdump, xxd, and more. The basic GNU debugger tool also comes default with Linux. Some problems can be solved with rudimentary knowledge of assembly language and decompilation; however, more advanced programs such as self-modifying malware requires in-depth knowledge of classes and structures. For now we will only work with the beginner files.

To start from the root-me homepage, click on Challenges >> Cracking


You are brought to a list of binary cracking challenges ordered by increasing difficulty. Select the first binary file labeled ELF-0 protection. On the next page click Start the challenge to download the file as ch1.zip. Unzip to ch1.bin and prepare for disassembly.



If not already done, ensure you have a software reverse engineering tool installed to aid your cracking. You can elect to use the default GNU tools, but I prefer a visual tool suite for solving these files. You will need all the help you can get while cracking more difficult binaries. This isn't to say objdump and file are useless. There is definitely a place for them and are great for initial "quick looks" as well as dynamic analysis.

I will use Ghidra to disassemble ch1.bin. Ghidra is an open-source reverse engineering framework developed by the National Security Agency (NSA). It provides support for a multitude of software architectures and instruction sets and ships with a wide array of analysis features. Ghidra can be downloaded here and requires Java Development Kit (JDK) version 11+ which can be downloaded here. Installation is pretty straight-forward once Java is set up. Unzip the ghidra package, navigate to the root folder (typically cd ghidra_ [$version-number]_PUBLIC_[$date-of-release]/ghidra_[$version-number]/), and run:

sudo ./ghidraRun.

[insert command line picture here]

You will run ghidraRun.bat if installing Ghidra on a Windows system. When you first run this command, Ghidra will ask you for the specific path to the JDK v11+ folder.
