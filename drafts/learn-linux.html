  Learn Linux Writeup

    Good afternoon Tim, this is my first writeup in a series of TryHackMe (THM) rooms.  This writeup is on an easy room called Learn Linux.  Since the first 42 tasks are accomplished by following simple directions, this writeup only focuses on the bonus task.  The bonus task is a Capture The Flag (CTF) challenge to find a root.txt document with the flag.  Everything needed to escalate privileges is covered in the 42 earlier tasks.

Enumeration

   All 42 tasks are completed on the current machine so a significant amount of enumeration can be done if you pay attention.  There are 5 normal users (shiba1, shiba2, shiba3, shiba4, nootnoot) and an admin user (root).  Each account has varying read/write permissions on different files.  While navigating around each users' home folder, I noticed a file named ll in /home/nootnoot.  I ran the ll command to find hidden files with .sudo_as_admin_successful listed.  This means user nootnoot has sudo permissions.  If only there was some way to log in as nootnoot...  During my enumeration, I found a lead on a file /var/log/test1234 in task 32.


I navigated to the file directory with cd /var/log and ran ll to view permissions.  The file test1234 allows read permissions for user shiba2, which is a user that was logged into earlier in the lesson.


Attempting to cat out the contents as shiba1 results in permission denied.


Simply switch user to shiba2 with the su command.  You should have shiba2's password from earlier in the room.  Once complete, cat the contents again to reveal nootnoot's password.  Log in as nootnoot to complete the initial foothold stage of this challenge.



Exploitation

   I tried sudo -l earlier as shiba1 but failed due to lack of sudo permissions.  Once nootnoot was compromised, I made a second attempt and acquired the below output.  This indicates nootnoot can run all commands as root.  In the words of Unix StackExchange: it's "security nirvana!"


A list of 120 Metasploit modules are returned.  Referring to the rapid7 page from earlier, we are looking for a module with ms08_067_netapi in its name, which is the 97th result.  Load it with the command use 97 or use exploit/windows/smb/ms08_067_netapi; either will work.  If you ever need to unset a module, type back.



Open module options with options to see if parameters need to be added.  Sure enough, it requires the target IP address.


Set the target with set RHOSTS 10.10.10.4 and confirm the result.



Once complete run the binary to activate /bin/bash in interactive mode!  Notice the username change.




Post-Exploitation

   Once root access is achieved, we need to find a flag.  Start searching for interesting files and directories.  Since Legacy is a Windows system, cmd commands are used to navigate the file system.  Type dir to view the contents and gain some situational awareness.





Thoughts

   Although an easy room, the Learn Linux challenge threw me off and I admittedly underestimated it when starting.  This challenge took me longer than I'm embarrased to admit, which was a little over an hour.  The main timesink was not having the initial piece for user compromise.  I knew exaclty which method to use for root privilege escalation, but reaching the necessary user took some time.  This was caused by insufficient enumeration early on.  Therefore, it is imperative that proper enumeration is done early on to avoid time sinks.  Everyone in the OSCP community repeats "enumerate, enumerate, enumerate" all the time and this is why.  Overall I rate this difficulty a 2/10 only because of prior experience in my escalation method.  A newer person may require a nudge on the final portion but good enumeration is something everyone at all levels benefit from.
