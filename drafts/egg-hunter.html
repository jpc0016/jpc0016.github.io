   Today's post involves writing Egg hunter shellcode.  Per the assignment instructions, this is a research oriented task for the purpose of integrating with other shellcode such as reverse shell.  A quick introduction is required since this is a new concept.

Concept

   Not all vulnerable programs are built alike.  They are often constrained by limited space to execute whatever is placed inside; therefore, it makes sense to split a payload into two stages.  Stage 1 searches memory space for Stage 2 and jumps to it.  Stage 2 is the intended payload to be executed.  The problem changes from a lack of space to a need for continuous execution.  Egg hunter is the solution to this problem.  Consider the comparison below (via Exploit-DB paper).



Both stacks are the same size except the right stack contains much less space after the input buffer (junk bytes).  The left stack has continuous memory space for shellcode execution so using stages is not necessary.  All 365 shellcode bytes fit so just let it rip like beyblades!  The right stack however, only has 190 bytes of space (where the question mark is) so where do the remaining 175 bytes go?
